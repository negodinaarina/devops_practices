# Лабораторная 1
## Задание
Настроить nginx по заданному тз:<br><br>

  1. Должен работать по https c сертификатом
  2. Настроить принудительное перенаправление HTTP-запросов (порт 80) на HTTPS (порт 443) для обеспечения безопасного соединения
  3. Использовать alias для создания псевдонимов путей к файлам или каталогам на сервере
  4. Настроить виртуальные хосты для обслуживания нескольких доменных имен на одном сервере
  5. Что угодно еще под требования проекта


## Ход работы
Чтобы выполнить задание в Nginx прежде всего нужно этот самый Nginx установить. Для Windows установка максимальна проста: нужно просто скачать архив с официального сайта.
Чтобы веб-сервер начал работу необходимо перейти в директорию программы и в терминале прописать `start nginx`

В задании требуется настроить обслуживание двух доменных имен на одном сервере по протоколу https, поэтому далее были выписаны самоподписанные сертификаты для обоих доменных имен
```
genrsa -out rootCA.key 2048
req -new -x509 -key rootCA.key -out rootCA.crt -days 1024
```
Аналогичным образом был создан сертфикат для второго домена.


Сервера были настроены на прослушивание 443 порта
```
server {
        listen       443 ssl;
        server_name  first_page.com www.first_page.com;
}
```

Для принудительного перенаправления с протокола HTTP на HTTPS были написаны обработчики для каждого домена на 80 порту. 
С помошью `return 301 https://first_page.com;` осуществляется перенаправление на протокол HTTPS, что обеспечивает безопасность соединения.
